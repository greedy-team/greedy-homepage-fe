# ADR 004. 모집 상태는 IS_RECRUITING 값 하나로 바꾼다

- 상태: 확정 (2026-07)
- 결정자: 4기 홈페이지 TF

## 맥락

모집은 1년에 몇 주지만 사이트는 1년 내내 열려 있어요. 평시에 마감일과 지원 버튼이 그대로 남아 있으면 방문한 사람이 멈춘 사이트라고 느껴요. 반대로 평시 화면이 준비되어 있으면 모집이 아닐 때도 동아리 소개 링크로 쓸 수 있어요.

## 결정

`src/shared/config/site.ts`의 `IS_RECRUITING` 값 하나가 세 곳을 같이 바꿔요.

| 위치 | 모집 중 | 평시 |
| --- | --- | --- |
| GNB 오른쪽 | 지원하기 (primary) | 모집 알림 받기 (ghost) |
| 히어로 | 모집 배지 + 마감일 + 지원하기 | 배지와 마감일 없음 + 다음 기수 알림 받기 |
| 하단 고정 CTA (모바일) | 랜딩에서만 보여요 | 없음 |

햄버거 메뉴 안의 지원하기도 평시에는 모집 알림 받기로 바뀌어요.

## 이유

- 흩어 놓으면 모집이 끝났을 때 하나쯤 놓치기 쉬워요. 값 하나면 놓칠 수가 없어요.
- 평시의 지원하기는 눌러도 아무것도 못 하는 죽은 버튼이에요. 대신 모집 알림 받기(구글폼으로 메일 수집)가 평시의 주 행동이에요. 모집이 열리면 알림 리스트가 초기 지원자 풀이 돼요.
- 버튼 자리는 유지하고 색만 바뀌어요(초록이면 모집 중, 테두리면 모집 전). 상태가 바뀌어도 화면이 낯설지 않아요.

## 결과

- 모집이 열리고 닫힐 때 바꾸는 것: `IS_RECRUITING` 값, 폼 주소 두 개. 끝이에요.
- 구글폼 첫 줄에는 수집 목적 한 문장을 적어요. 하단 고정 CTA는 모집 중에도 랜딩에서만 써요. 다른 화면까지 따라가면 소개보다 광고로 읽혀요.
