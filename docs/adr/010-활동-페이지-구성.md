# ADR 010. 활동 페이지 구성

- 상태: 확정 (2026-07)
- 결정자: 4기 홈페이지 TF

## 맥락

활동 페이지는 행사·엠티·데모데이처럼 함께한 순간의 기록이에요. 스터디·프로젝트가 "무엇을 만들었나"라면 활동은 "어떻게 지냈나"를 보여줘요. 사진이 콘텐츠의 중심이고, 시간이 정보의 축이에요.

## 결정

### 목록

- **타임라인 축** - 날짜와 도트·선으로 시간축을 그려요. 시간축이 곧 정보 구조라, 도트와 선이 스캔 경로를 만들어 목록을 훑는 인지 비용을 낮춰요.
- **필터도 배지도 없어요** - 언제는 날짜 축이, 무엇인지는 제목이 이미 말해줘요. 기수 탭은 날짜와 겹치는 정보였고, 종류 배지는 "엠티", "대동제 부스"처럼 제목과 겹쳐요. 종류를 빼면 활동을 올릴 때마다 분류를 정하는 부담도 사라져요.
- **카드에는 대표 사진 하나만** - 사진을 펼쳐 보여주는 건 상세 갤러리의 일이에요. 카드가 낮아야 타임라인을 훑기 쉬워요.
- **더 보기 없이 전부** - 활동은 수십 개 규모라 자를 이유가 없고, 기수 탭과 종류 칩이 이미 목록을 나눠줘요. 몇 년치가 쌓여 길어지면 그때 다시 판단해요.

### 상세

- **헤더 위계** - 제목 → 날짜와 장소 → 참여 멤버. 무엇을, 언제 어디서, 누구와 했는지라는 질문 순서 그대로예요.
- **아바타 스택** - 참여 멤버를 얼굴로 보여줘서 활동과 사람을 연결해요. 멤버 프로필로 가는 진입점이자, 기록이 이력이 되는 구조의 시작점이에요.
- **대표 사진** - 본문 앞에 16:9 한 장으로 맥락을 먼저 줘요. 시각 정보가 텍스트를 읽기 전의 앵커 역할을 해요.
- **갤러리 두 상태** - 사진이 4장 이하면 전부 펼치고, 넘으면 3장 + 모두 보기 타일로 줄여요. 첫 화면 높이를 지켜서 본문과 이전 다음 내비의 도달률을 지켜요. 누르면 라이트박스로 크게 봐요 ([ADR 없음, #26](https://github.com/greedy-team/greedy-homepage-fe/issues/26)).
- **복귀 링크와 이전 다음** - 목록으로 돌아가는 비용과 다음 콘텐츠로 넘어가는 비용을 모두 낮춰요. 이탈 대신 순환을 만들어요. 이전 다음은 데스크톱에서만 보여요.

## 코드 배치

- 활동 페이지에서만 쓰는 구획이라 `app/activities/_sections`, `app/activities/[id]/_sections`에 라우트와 같이 둬요 ([ADR 005](005-폴더-구조-축소형-FSD.md)).
- 필터가 없어서 목록은 전부 서버 렌더예요. client는 갤러리 열림 상태 하나뿐이에요.
- 활동은 서버에서 올 데이터라 `entities/activity`에 둬요 ([ADR 003](003-콘텐츠-코드-분리.md)). 목록은 `ActivitySummary`, 상세는 `Activity`로 나눠요. 명세 확정 전까지 정적 데이터고, 확정되면 함수 안만 fetch로 바꿔요 ([ADR 002](002-데이터는-빌드타임-ISR.md)).
- 참여 멤버는 `entities/member`와 이름으로 맞춰 페이지(app)에서 이어요. entity끼리는 참조하지 않아요.
