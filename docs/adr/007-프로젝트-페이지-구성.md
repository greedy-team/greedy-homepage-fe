# ADR 007. 프로젝트 페이지 구성

- 상태: 확정 (2026-07)
- 결정자: 4기 홈페이지 TF

## 맥락

프로젝트 페이지는 그리디가 실제로 만든 것들을 보여줘요. 지망 부원과 다른 학우, 함께 일할 사람이 "무슨 문제를 어떻게 푼 서비스인지"를 빨리 파악하고, 코드까지 보고 싶으면 저장소로 넘어갈 수 있어야 해요.

## 결정

목록(`/projects`)과 상세(`/projects/[id]`) 두 화면으로 둬요.

### 목록

- 전 기수 프로젝트를 최신 기수부터 카드 그리드로 보여줘요.
- **기수 필터**(전체 + 각 기수)로 걸러요. 프로젝트가 없는 기수(지금은 4기)를 고르면 준비 중임을 알리는 `EmptyState`를 보여줘요.

### 상세 (위에서 아래로)

1. 헤더 - 뒤로가기 · 기수 배지 · 이름 · 링크 버튼
2. 대표 이미지
3. 한 줄 요약 → 어떤 문제를 풀었나요 → 주요 기능 → 기술 스택
4. 화면 (추가 이미지, 넘치면 +n)
5. 함께 만든 사람들 (FE/BE/디자인, 외부 기여자 포함)
6. 이전 / 다음 (데스크톱만)

### 저장소 링크

서비스(있으면) · 프론트 GitHub · 백엔드 GitHub를 헤더 버튼으로 둬요. 없는 링크는 버튼을 그리지 않아요. 데스크톱은 제목 옆에, 모바일은 이미지 아래 전체폭으로 내려가요.

## 이유

- **글 먼저, 화면 나중.** 히어로 이미지로 눈을 끈 뒤 요약·문제·기능으로 뭘 하는 서비스인지 이해시키고, 화면은 그 근거로 뒤에 보여줘요. 화면을 먼저 몰아 보여주면 뭔지도 모르고 스크롤하게 돼요.
- **저장소는 헤더에.** 스택 옆에 두는 안도 있었지만, 코드를 찾는 사람은 개발자라 주요 행동 옆에서 바로 가는 게 발견성이 좋아요. 프론트·백엔드가 갈린 건 버튼 이름으로 구분해요.
- **기수 필터는 넣되 빈 기수를 대비해요.** 현재 기수는 아직 프로젝트가 없을 수 있어서, 그 탭을 고르면 빈 화면 대신 준비 중 안내를 보여줘요.
- **이전/다음은 데스크톱만.** 모바일 시안에는 없어요. 좁은 화면에서 카드 두 장이 세로로 쌓이면 길이만 늘어요.

## 코드 배치

- 프로젝트 페이지에서만 쓰는 구획이라 `app/projects/_sections`, `app/projects/[id]/_sections`에 라우트와 같이 둬요 ([ADR 005](005-폴더-구조-축소형-FSD.md)).
- 프로젝트는 서버에서 오는 데이터라 `entities/project`에 둬요 ([ADR 003](003-콘텐츠-코드-분리.md)). 명세가 확정되기 전까지 `api.ts`가 정적 데이터를 반환하고, 확정되면 함수 안만 fetch로 바꿔요 ([ADR 002](002-데이터는-빌드타임-ISR.md)).
- 목록 카드는 요약만 필요해서 `ProjectSummary`, 상세는 `Project`로 나눠요. 목록이 상세의 무거운 필드까지 안 들고 다녀요.
- 필터 상태만 client(`ProjectList`)이고, 나머지는 서버에서 렌더해요. 페이지 문구와 빈 상태 안내는 `app/projects/_sections/content.ts`에 둬요.
