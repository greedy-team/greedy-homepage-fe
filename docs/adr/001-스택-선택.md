# ADR 001. Next.js와 Tailwind v4, cva를 쓴다

- 상태: 확정 (2026-07)
- 결정자: 4기 홈페이지 TF

## 맥락

HTML 프로토타입으로 화면과 흐름을 검증했고, 실제 구현 스택을 정해야 했어요. 이 사이트는 몇 년 뒤의 기수가 이어받아야 해서, "지금 편한 것"보다 "물려주기 쉬운 것"이 기준이었어요.

## 결정

- **Next.js (App Router)**: 페이지 대부분이 정적인 홍보, 아카이브 성격이라 빌드 타임 렌더링과 궁합이 좋아요. 프론트 스터디가 React 기반이라 후배들이 이어받기도 쉬워요.
- **Tailwind CSS v4**: 디자인 토큰(색, 글자, 간격, 라운드)을 `@theme` CSS 변수로 선언해서 한곳에서 관리해요. 스타일이 마크업 옆에 있어서 컴포넌트를 통째로 이해하기 쉬워요.
- **cva**: 버튼처럼 배리언트가 있는 컴포넌트를 `variant`, `size` 프롭으로 선언적으로 관리하기 위해 써요.
- **Storybook**: 디자인 시스템 컴포넌트의 상태와 배리언트를 눈으로 확인하고 문서화하는 도구예요.
- **styled-components는 쓰지 않아요**: 런타임 비용이 있고, 토큰과 배리언트 관리가 위 조합보다 번거로워요.

## 결과

- 디자인 토큰의 진실원천은 코드(`@theme`)예요. 토큰에 없는 값은 쓰지 않아요.
- 공용 컴포넌트에는 스토리를 함께 만들어서, 상태별 모습을 페이지에 붙이기 전에 확인할 수 있어요.
