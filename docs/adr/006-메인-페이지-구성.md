# ADR 006. 메인(랜딩) 페이지 구성

- 상태: 확정 (2026-07)
- 결정자: 4기 홈페이지 TF

## 맥락

랜딩은 사이트 첫 화면이에요. 모집 때는 지원 창구, 평시에는 소개 페이지로 써요([ADR 004](004-모집-상태-스위치.md)). 처음 온 사람이 그리디가 뭘 하는 곳인지, 지금 뭘 할 수 있는지 빨리 알 수 있어야 해요.

## 결정

위에서 아래로 6개 구획을 이 순서로 둬요.

1. **히어로** - 한 줄 소개 + 모집 상태 CTA(지원/알림). 그린 배경과 엠블럼.
2. **스탯 밴드** - 누적 멤버, 기수, 트랙, 프로젝트 수.
3. **그리디의 한 학기** - 스터디 → 리뷰 → 프로젝트 → 데모데이.
4. **활동** - 함께한 활동 사진.
5. **프로젝트** - 실제로 만든 것들.
6. **가치 밴드** - OPEN, KINDNESS, TOLERANCE, HUMAN COMMUNITY.

모집 상태는 `IS_RECRUITING` 하나로 히어로 CTA·배지·안내문과 모바일 하단 CTA를 같이 바꿔요([ADR 004](004-모집-상태-스위치.md)).

## 이유

- 소개 → 뭘 하는지 → 결과물 순서예요. 스탯과 한 학기로 어떤 곳인지 먼저 알려주고, 활동·프로젝트로 실제 결과를 보여줘요.
- 가치 밴드는 맨 끝에 뒀어요. 활동·프로젝트를 본 뒤라야 가치 선언이 공허하지 않아요.
- 초록 버튼(primary)은 히어로 CTA 하나뿐이에요. 나머지는 더보기 텍스트 링크라, 눌러야 할 게 헷갈리지 않아요.
- 히어로와 가치 밴드가 둘 다 그린 배경이라 페이지 위아래를 감싸요. 사이 구획은 흰 배경.

## 코드 배치

- 홈에서만 쓰는 섹션이라 `app/_sections`에 라우트와 같이 둬요([ADR 005](005-폴더-구조-축소형-FSD.md)). 다른 페이지에서 쓰게 되면 그때 `widgets`로 옮겨요.
- 서버에서 오는 데이터(활동·프로젝트)는 `entities/<도메인>/api`. 정적 문구와 스탯은 `app/_sections/content.ts`.
- 스탯은 백엔드가 아니라 기수마다 손으로 갱신하는 값이라, 문구처럼 `content.ts`에 함께 뒀어요.
- `page.tsx`는 데이터 불러서 섹션에 넘기는 조립만 해요. 구획 추가나 순서 변경은 `page.tsx`만 고치면 돼요.
