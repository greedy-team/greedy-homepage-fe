# ADR 003. 콘텐츠는 코드와 분리해 src/data에 둔다

- 상태: 확정 (2026-07)
- 결정자: 4기 홈페이지 TF

## 맥락

이 사이트의 유지보수 대부분은 코드 수정이 아니라 콘텐츠 갱신이에요. 새 기수 커리큘럼, 문의 링크 변경 같은 일이요. 몇 년 뒤의 운영진이 처음 열어 봐도 고칠 수 있어야 해요.

## 결정

- 하드코딩 콘텐츠(스터디 커리큘럼, 홈 문구, 가치 선언 등)는 전부 `src/data`의 데이터 파일로 분리해요.
- 컴포넌트 안에 문구와 데이터를 직접 쓰지 않아요.
- 사이트 전역 상수(모집 상태, 링크)는 `src/lib/site.ts` 한 곳에 둬요.

## 이유

- "5기 커리큘럼으로 바꾸기"가 데이터 파일 하나 수정으로 끝나요.
- 콘텐츠 수정 PR과 코드 수정 PR이 분리되어 리뷰가 쉬워요.
- 콘텐츠가 한곳에 모여 있으면 문구 톤과 표기(해요체, 대시 규칙)를 일괄 점검할 수 있어요.

## 결과

- 콘텐츠 갱신용 이슈 템플릿이 있어요.
- 백엔드 API로 가는 도메인(프로젝트, 멤버, 활동)이 늘어도 이 원칙은 같아요. 데이터는 언제나 컴포넌트 밖에서 와요.
